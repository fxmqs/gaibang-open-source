# 弟子创建指南

## 一、弟子结构

每个弟子包含以下文件：

```
弟子目录/
├── SOUL.md           # 人设（必选）
├── task.md           # 任务说明（必选）
├── agent.json        # Agent 配置（可选）
└── rules.md          # 规则说明（可选）
```

## 二、创建流程

### Step 1: 创建目录
```bash
mkdir -p agents/{分堂}/{弟子代号}
```

### Step 2: 创建 SOUL.md
参考模板编写人设文档

### Step 3: 创建 task.md
参考模板编写任务说明

### Step 4: 创建 agent.json (可选)
```json
{
  "id": "{分堂}/{弟子代号}"
}
```

### Step 5: 添加到验收体系
根据任务类型，在对应验收使的 task.md 中添加验收标准。

## 三、最佳实践

1. **职责单一** - 每个弟子只负责一类任务
2. **输出规范** - 统一文件命名和 Front Matter
3. **丐帮签名** - 所有汇报必须有开头和结尾签名
4. **验收标准** - 明确可量化的验收指标

详见完整文档。
