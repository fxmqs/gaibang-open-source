# 验收体系设计

## 一、概述

验收体系是丐帮质量管控的核心，确保所有任务产出符合标准。

## 二、六大验收使

### 1. 数据验收使
**职责**: 验收数据采集类任务

| 检查项 | 要求 |
|--------|------|
| Front Matter | tags/created/author/source 完整 |
| 丐帮签名 | 开头+结尾 |
| 数据量 | >= X 条（按任务要求） |
| 摘要 | >=200字/条 |
| 点评 | >=200字/条 |

**适用弟子**: 风媒、硅谷通、开源客、模型侠、舆情通、视界通、Moltbook专员

### 2. 代码验收使
**职责**: 验收技能开发类任务

| 检查项 | 要求 |
|--------|------|
| 代码运行 | npm install 成功 |
| 功能测试 | 单元测试 >=80% |
| 代码质量 | 无 lint 错误 |
| 文档完整 | README + 使用示例 |

**适用弟子**: 匠人、鲁班

### 3. 环境验收使
**职责**: 验收环境安装类任务

| 检查项 | 要求 |
|--------|------|
| 安装日志 | 软件版本符合要求 |
| 环境可用 | 命令能正常执行 |
| 自测通过 | 自检脚本通过 |
| 磁盘空间 | >10GB 可用 |

**适用弟子**: 移动使、重启使

### 4. 整改验收使
**职责**: 验收整改修复类任务

| 检查项 | 要求 |
|--------|------|
| 问题修复 | 按令牌要求逐项修复 |
| Front Matter | 完整 |
| 丐帮签名 | 开头+结尾 |
| 内容完整 | 摘要、点评 >=200字 |

**适用弟子**: 所有弟子

### 5. 调研验收使
**职责**: 验收调研分析类任务

| 检查项 | 要求 |
|--------|------|
| 报告结构 | 文档结构完整 |
| 分析深度 | >=500字分析 |
| 建议可行 | 给出可执行建议 |
| 丐帮签名 | 开头+结尾 |

**适用弟子**: 调研监控使、评书先生

### 6. 运维验收使
**职责**: 验收系统运维类任务

| 检查项 | 要求 |
|--------|------|
| 服务状态 | 进程/服务正常运行 |
| 日志正常 | 无 ERROR 级别日志 |
| 监控指标 | CPU/内存/磁盘在正常范围 |
| 自愈结果 | 故障恢复成功 |

**适用弟子**: 重启使、系统监控使

## 三、验收流程

```
任务完成
    ↓
Step 1: 传令使初验（路径+文件存在）
    ↓
Step 2: 按类型分发验收使
    ↓
Step 3: 验收使逐项验收
    ↓
    ✅ 通过 → 令牌闭环
    ❌ 失败 → 返回整改
```

## 四、验收模板

每个验收使有统一的验收报告模板：

```markdown
## 验收报告

### 令牌: LINGPai_20260215_XXXX
### 文件: 文件路径

| # | 检查项 | 要求 | 结果 |
|---|--------|------|------|
| 1 | 检查项1 | 要求 | ✅/❌ |
| 2 | 检查项2 | 要求 | ✅/❌ |

**验收人**: [验收使名称]
**日期**: YYYY-MM-DD
**结果**: ✅ 通过 / ❌ 不通过
```

## 五、验收统计

| 指标 | 计算方式 |
|------|----------|
| 验收通过率 | 通过数 / 总验收数 |
| 平均验收时间 | (验收完成时间 - 任务完成时间) / 验收数 |
| 整改次数 | 验收失败后整改的次数 |
